# 群聊线下功能开发计划 (Group Offline Mode Plan)

- [x] **步骤一：探索与确定单人线下逻辑**
  - 查看现有的“单聊线下模式”是如何触发的（按钮位置、UI表现、绑定的事件）。
  - 了解线下模式状态是如何在系统中保存的。
  - **【心得记录】**：单聊的线下模式由 ID 为 `offline-mode-toggle` 的开关控制。切换时会在 `modules/init-event-bindingsA.js` 中触发事件，将状态保存到 `chat.settings.isOfflineMode` 中。底层通过判断此状态决定提示词分支。要在群聊中实现一模一样的功能，我们只需要在群聊设置界面也放一个开关，把状态写进类似 `group.settings.isOfflineMode` 中即可。

- [x] **步骤二：群聊UI入口添加**
  - 在群聊界面或群聊设置中，添加与单聊完全一致的“线下模式”开关/按钮。
  - 确保UI样式复用单聊的设计，不引入新的版块。

- [x] **步骤三：群聊线下状态管理**
  - 修改状态记录逻辑，使每个群聊（基于 groupId）都能独立保存其“线下模式”的开启/关闭状态。
  - 处理切换开关时的状态持久化和界面反馈。

- [ ] **步骤四：群聊 Prompt（提示词）动态注入**
  - 修改 AI 请求的构建逻辑 (`ai-response.js` / `prompt-manager.js` 等)。
  - 当检测到当前群聊处于线下模式时，在系统提示词中动态注入：“注意：你们现在处于线下聚会状态。现场有[用户]、[群成员A]、[群成员B]等...”
  - 确保 AI 能够理解并产生符合线下群体聚会场景的对话和动作。

- [ ] **步骤五：测试与收尾**
  - 验证开关功能的稳定性。
  - 测试 AI 在群聊线下模式下的回复表现，确认是否和单人线下的代入感一致。
